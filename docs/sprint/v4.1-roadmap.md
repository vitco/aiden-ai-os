## Engineering debt from Phase 16–21 audits

- **Provider fallback on empty-response.** Phase 23.4a-fix2 added a per-turn empty-response retry guard: when the Codex backend (ChatGPT Plus OAuth) returns a completion with no content and no tool calls, Aiden injects a corrective system message and retries once before surfacing honest failure. Rapid back-to-back queries in the same session can still produce two empty completions in a row — the user gets the honest failure and has to `/exit` for a fresh session. v4.1 fix: when the empty-response cap exhausts, auto-switch to the next provider in the configured fallback chain (Together AI / Groq / Ollama) instead of surfacing failure to the user. Counter visible at `/doctor` as `[empty-response] detected=N retried=N recovered=N`.
- **Intent-based enforcement arming.** Phase 23.1 mechanical skill-required-tool enforcement arms only when `skill_view` returns a payload with `required_tools`. Same-session reuse of an already-loaded skill bypasses the guard: the model executes the skill's flow without ever calling `skill_view` again, so the tracker never arms, so the missing-tool retry never fires. Captured in Phase 23.3 smoke (run 2: media-search reused, model asked instead of opened, enforcement silent). v4.1 fix: derive arm-trigger from user intent (regex/classifier on the user message + skills_list catalog) so arming is independent of `skill_view` re-invocation. Out of scope for v4.0 launch — current prompt-side anti-asking (Phase 23.3) is the temporary cover.
